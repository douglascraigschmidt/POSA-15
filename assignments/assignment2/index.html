Concurrent Ping-Pong Application (P1 - Assignment 2)

Overview of the Concurrent Ping-Pong Application Assignment

The purpose of this assignment is to give you experience using Android HaMeR framework classes to develop a multi-threaded app that uses message passing to alternate printing "ping" and "pong" to the user display. In particular, this assignment involves the use of HaMeR classes like HandlerThread, Handler, Looper, and (implicitly) MessageQueue. You'll also need to use a Java CyclicBarrier to ensure that HandlerThreads are fully initialized before the concurrent ping-pong algorithm begins. No other Java synchronization mechanisms are needed for this assignment.

In addition to the lectures on Android we'll present in class during week 2, the following resources may be helpful in completing portions of this assignment:

  • Android Looper
  • Overview of Android Handler
  • Sending and Handling Messages with Android Handler
  • Java Barrier Synchronization (around the 40:00 mark starting with slide 86)

Skeleton Code

Skeleton code for this assignment is available here. Pull the appropriate skeleton code in your repository, read it carefully, and complete the "TODO" markers. In addition to filling in the "TODO" code snippets, I also recommend looking carefully at the code provided in the MainActivity.java class, which illustrates other interesting uses of the HaMeR framework.

Note that there are no automated unit tests for this assignment. Instead, you'll know the program is working when you can successfully generate "ping" and "pong" requests in the proper order. Make sure that your program works properly for both correct and incorrect input entered by the user.

Concluding Remarks

This assignment is designed to help you deepen your familiarity with concurrent programming of the Android HaMeR framework. It's more complicated than assignment1, so make sure you start early and participate in Virtual Office Hours if you have questions or run into problems.
